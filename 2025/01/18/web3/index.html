攻防世界 Web_python_template_injection
打开题目提示python template injection—->Python 模板注入
以下出现的[[……]]为”中括号中括号……中括号中括号”
打开题目提示 “python template injection”，即 Python 模板注入。这是一种在 Web 应用程序中利用 Python 模板引擎漏洞的攻击技术。涉及的模板引擎主要有：
Jinja2：广泛应用于 Flask 等 Web 框架。它功能强大，支持变量替换、控制结构（如循环、条件判断）以及宏定义等。
Django 模板引擎：内置于 Django 框架中，提供了一种安全且简洁的方式来生成动态网页。它具有独特语法，用于将视图层的数据和展示层的模板相结合。
攻击原理
当 Web 应用接收用户输入，并将其直接嵌入到模板中进行渲染，且未对输入进行严格验证与过滤时，就会引发问题。模板引擎提供诸多功能以实现动态内容生成，攻击者可利用这些功能达成恶意目的。比如在 Jinja2 中，通过[[self.init.globals[‘os’].popen(‘恶意系统命令’).read() ]]，攻击者能借助模板引擎机制访问全局命名空间，获取os模块来执行恶意系统命令。在 Jinja2 模板引擎中，[[]]不仅可以传递变量，还可以执行一些简单的表达式。
我们可以输入[[数字 + 数字]]来判断有无模板注入，例如：http://61.147.171.105:65310/2。如果显示 “URL http://61.147.171.105:65310/2 not found”，则表明存在模板注入。此时，我们可以通过[[config]]查看全局变量，通过[[‘’.class.mro[2].subclasses()]]寻找可用引用。
